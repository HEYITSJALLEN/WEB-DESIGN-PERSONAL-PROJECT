Why do we need a viewport meta tag?

The viewport meta tag controls how a web page is displayed on different screen sizes, especially on mobile devices. By default, browsers render web pages at a fixed width, which can make sites appear zoomed out and hard to read on smaller screens. The viewport tag allows you to define a scalable, responsive experience by setting parameters like width and initial scale.

When you select the same tag twice, and in one of them you say "flex-direction: column;" and the other says "flex-direction: row;", which one takes priority?

When CSS properties conflict for the same element, the "cascading" nature of CSS dictates that the last rule defined in the stylesheet (or inline style) takes priority, given equal specificity.

How do you change the CSS for a web page displayed on a phone?

To target specific screen sizes, like those for phones, you can use CSS media queries. Media queries allow you to apply styles only when certain conditions are met, such as a maximum width.
